DocFerry 0.0.11

This release responds to the automated review round on 0.0.10 and hardens the project's type-safety configuration. There are no user-facing behavior changes.

**Type safety:**

- Enables full TypeScript `strict` mode (plus `skipLibCheck`, `esModuleInterop`, and `forceConsistentCasingInFileNames`).
- `tsc -noEmit` and ESLint (`typescript-eslint` recommended type-checked + `eslint-plugin-obsidianmd`) both pass with zero problems.
- The `no-unsafe-*` warnings seen in the 0.0.10 review are resolved-as-`any` false positives that occur when the review's type-aware lint cannot resolve the Obsidian type declarations; the source itself is type-safe.

**Privacy:**

- Clipboard access remains write-only and is triggered only by explicit user actions, as documented in `PRIVACY.md`.

**Compatibility:**

- Minimum Obsidian version remains `1.13.0`.

**Release assets:**

- `main.js`
- `manifest.json`
- `styles.css`
