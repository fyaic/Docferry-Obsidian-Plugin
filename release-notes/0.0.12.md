DocFerry 0.0.12

This release makes the repository's type setup resolvable from the repo root so the Obsidian automated review can resolve the `obsidian` type declarations. There are no user-facing behavior changes.

**Automated review / type resolution:**

- Adds repository-root `package.json`, `package-lock.json`, `tsconfig.json`, and `eslint.config.mjs`.
- These let the review environment install and resolve `obsidian` types from the repo root even though the plugin source lives under `plugin/`.
- This clears the `@typescript-eslint/no-unsafe-*` warnings, which appeared only because the audit could not resolve Obsidian's types — the plugin source already lints clean locally (`tsc -noEmit` and ESLint both pass with zero problems).
- `obsidian` (1.13.0) and its `@codemirror/state` (6.5.0) / `@codemirror/view` (6.38.6) peers are pinned to exact versions so a clean `npm install` resolves without peer-dependency conflicts.

**No behavior changes:**

- The built `main.js` is unchanged in content; commands, settings, and publish/import/share flows are identical.

**Compatibility:**

- Minimum Obsidian version remains `1.13.0`.

**Release assets:**

- `main.js`
- `manifest.json`
- `styles.css`
